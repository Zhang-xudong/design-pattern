UML图讲解

1.概述

学习设计模式避免不了接触大量的UML图，并且UML图并不限于只为设计模式而存在，它在其他地方也应用广泛。所以在正式学习设计模式之前我们必须对UML图有个大体的认识。

2.分类讨论

2.1 类与接口

首先上图中的那个“动物”矩形框代表了一个类。类图分三层，第一层显示类的名称，如果是抽象类就用斜体字表示。第二层是类的属性，第三层是类的方法。注意前面的符号“+”表示public，“-”表示private，“#”表示protected。

然后注意左下角的“飞翔”，它表示一个接口图，玉类图的主要区别是顶端有<>标识。第一行是接口名称，第二行是接口方法。接口还有另一种表示方法，俗称棒棒糖表示法，如图中最下方的“唐老鸭”就是实现了“讲人话”的接口。

interface IFly{
    void fly();
}

interface ILanguage{
    void speak();
}

2.2 类与类、类与接口之间的关系

2.2.1 继承关系

我们注意一下上图中动物、鸟、鸭子、唐老鸭之间的关系符号，它们相互之间是继承关系，继承关系就用空心箭头+实线来表示。

2.2.2 实现接口

图中列出的几种鸟类中，大雁是最能飞的，其实现了飞翔接口，实现接口用空心箭头+虚线表示。

//继承动物类
class Bird extends Animal{

}

//实现飞翔接口
class WideGoose implements IFly{

}

2.2.3 关联关系

我们在看企鹅和气候两个类，企鹅和气候有很大的关联性，只能在严寒的气候下生存，所以“企鹅”需要知道“气候”的变化，需要“了解”气候的规律。当一个类知道另一个类时，可以用关联关系（association）来表示，关联关系用实线箭头来表示。

class Penguin extends Bird{
    //在企鹅类Penguin中，引用到气候Climate对象。企鹅需要了解气候
    private Climate climate;
}

2.2.4 聚合关系

我们再来看看大雁与雁群这两个类，大雁是群居动物，每只大雁都隶属于一个雁群，而一个雁群可以有多只大雁，它们之间就满足聚合关系（Aggregation）。聚合关系体现的是一种弱的‘拥有’关系，体现的是A对象可以包含B对象，但B对象不是A对象的一部分。聚合关系用空心的菱形+实线箭头来表示。

//雁群类
class WideGooseAggregation{
    //在雁群WideGooseAggregation类中，有大雁数组对象arrayWideGoose
    private WideGoose[] arrayWideGoose;
}

2.2.5 合成（组合）关系

相比较于上面提到的“聚合关系”是一种弱的拥有关系，合成（Composition,也有翻译成“组合”的）是一种强的‘拥有’关系，体现了严格的部分和整体的关系，部分和整体的生命周期是一样的。在这里鸟和翅膀是合成关系，因为它们是整体和部分的关系，而且它们的生命周期是一样的。合成关系用实心的菱形+实线箭头来表示。另外，合成关系的连线两端有数字‘1’和数字‘2’，这被称为基数。表示这一端的类可以有几个实例，很显然一只鸟应该有两个翅膀。如果一个类可以有无数个实例，就用‘n'来表示。关联关系、聚合关系也可以有基数。

class Bird{
    private Wing wing;
    public Bird(){
        //在鸟Bird类中，初始化时，实例化翅膀Wing,它们之间同时生成
        wing = new Wing();
    }
}

2.2.6 依赖关系

动物有几大特征，比如新城代谢、能繁殖。而动物要有生命力就需要有水、氧气以及食物等。也就是说，动物依赖于水和氧气，它们之间是依赖关系（Dependency），用虚线箭头来表示。

abstract class Animal{
    //新陈代谢方法
    public metabolism(Oxygen oxygen, Water water){

    }
}
