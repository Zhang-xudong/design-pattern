简单工厂模式

1.面向过程思想

假如让你实现一个计算器的程序，功能就是简单的输入两个数字和运算符然后得到对应的结果。如果对面向对象思想没有深刻理解，也没有系统化的学习过设计模式，我们写出来很可能是如下这样：

初始版：

public class CalculaterProgram {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println("请输入数字A:");
        int numA = scanner.nextInt();
        //添加这行代码主要是为了吸收结束符\n，这是nextInt()方法和nextLine()方法的一些细节
        scanner.nextLine();
        System.out.println("请输入运算符号(+、-、*、/):");
        String operation = scanner.nextLine();
        System.out.println("请输入数字B:");
        int numB = scanner.nextInt();
        //存放最终结果
        int result = 0;

        if("+".equals(operation)){
            result = numA + numB;
        }
        if("-".equals(operation)){
            result = numA - numB;
        }
        if("*".equals(operation)){
            result = numA * numB;
        } 
        if("/".equals(operation)){
            result = numA / numB;
        }

        System.out.println("计算结果为:" + result);
    }
}

以上代码是典型的面向过程思想，整体来说没什么大毛病，主要有两点需要注意：

  1. if判断分支的写法，这样写的话意味着每个条件都要做判断，等于计算了三次无用功。
  2. *关于运算符和除法运算，如果用户输入的除数为0或者运算符不是对应的加减乘除，程序就会崩掉。

改进版：

public class CalculaterProgram {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        try {
            System.out.println("请输入数字A:");
            int numA = scanner.nextInt();
            //添加这行代码主要是为了吸收结束符\n，这是nextInt()方法和nextLine()方法的一些细节
            scanner.nextLine();
            System.out.println("请输入运算符号(+、-、*、/):");
            String operation = scanner.nextLine();
            System.out.println("请输入数字B:");
            int numB = scanner.nextInt();
            //存放最终结果
            int result = 0;

            switch (operation){
                case "+":
                    result = numA + numB;
                    break;
                case "-":
                    result = numA - numB;
                    break;
                case "*":
                    result = numA * numB;
                    break;
                case "/":
                    //除数不可为0
                    if(numB != 0){
                        result = numA / numB;
                    }else {
                        System.out.println("除数不可为0！");
                    }
                    break;
            }
            System.out.println("计算结果为:" + result);
        } catch (Exception e) {
            System.out.println("你的输入有误" + e.getMessage());
        }
    }
}

以上代码虽然解决了问题，但它还是典型的面向过程思想。所有编程初学者都会有这样的问题，就是碰到问题就之绝地用计算机能够理解的逻辑来描述和表达待解决的问题即具体的求解过程。这其实是用计算机的方式思考，比如计算器这个程序，要求输入两个数和运算符号，然后根据运算符号判断选择如何运算，最后得到结果，这本生没有错，但这样的思维却使得我们的程序只为满足实现当前的需求，程序不容易维护，不容易扩展，更不容易复用。从而达不到高质量代码的要求。

2.面向对象思想

我们举个简单的例子来说明一下。曹操创作 的《短歌行》。话说三国时期，育操带领百万大军攻打东吴，大军在长汪赤壁驻扎，军船连成一片，眼看就要灭 掉东吴，统一天下，四操大悦，于是大宴众文武，在酒席间，flf操诗性大发，不觉吟诗：‘喝酒唱歌，人 生真爽。……’。众文武齐呼：‘压相好诗！’于是一臣子速命印刷工匠刻版印刷，以便流传天下。

样张出来给酋操一看，曹操感觉不妥，说道：‘喝与唱，此话过俗，应改为‘对酒当歌’较好！ 于是就命工匠重新来刻。工匠眼看连夜刻版之工，彻底白费，心中叫苦不迭。只得照办。

样张再次出来请曹操过目，曹操细细一品，觉得还是不好，说：‘人生真爽太过直接，应改问语才 够意境，因此应改为‘对酒当歌．人生儿何？……’当臣转告工匠之时，工匠晕倒……！

三国时期没有发明活字印刷，所以当需要改动的时候整块印刷版都得丢掉把改动的内容可在新板上。而有了活字印刷以后，像上面的一系列改动只需修改四个字就可以，其他的都可以复用。

第一，要改，只需更改要改之字，此为可维护；第二，这些字并非用完一次就没用了，完全可以在后来的印刷中重复使用，此为可复用；第三，这首诗如果需要加字，只需另刻字加入即可，这是可扩展；第四，字的排列可能是横排也可能是竖排，此时只需要将活字做相应的移动就可以满足排列需求，这是灵活性好。面向对象的编程思想就是通过封装、继承、多态这些特性把程序的耦合度降低，而进一步通过使用设计模式可以使程序更加灵活、容易修改、并且易于复用。

3.利用面向对象思想改造代码

面向对象的三大特性就是封装、继承和多态，我们试一试怎样利用这三个特性将上面的代码做进一步改造。

3.1封装

上面我们利用面向过程思想写的代码其实是耦合度很高的，应该把“业务逻辑”和“显示逻辑”分开，让他们之间的耦合度降低，只有分离开才可以达到容易维护和扩展的目的。我们先从封装的角度着手改造：

Operation计算类

public class Operation {
    public static double getResult(double numA,double numB,String operate){
        double result = 0d;
        switch (operate){
            case "+":
                result = numA + numB;
                break;
            case "-":
                result = numA - numB;
                break;
            case "*":
                result = numA * numB;
                break;
            case "/":
                result = numA / numB;
                break;
        }
        return result;
    }
}

客户端代码

public class CalculateClient {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        try {
            System.out.println("请输入数字A:");
            double numA = scanner.nextInt();
            //添加这行代码主要是为了吸收结束符\n，这是nextInt()方法和nextLine()方法的一些细节
            scanner.nextLine();
            System.out.println("请输入运算符号(+、-、*、/):");
            String operation = scanner.nextLine();
            System.out.println("请输入数字B:");
            double numB = scanner.nextInt();
            //存放最终结果
            double result = 0;
            //直接调用Operation类即可
            result = Operation.getResult(numA,numB,operation);

            System.out.println("计算结果为:" + result);
        } catch (Exception e) {
            System.out.println("你的输入有误" + e.getMessage());
        }
    }
}

这样就把业务和界面分离了，业务端的代码Operation类可以达到复用的目的。

3.2继承

我们上面通过封装实现了业务和界面的分离，让业务代码做到了可复用，但是它能做到很灵活的可修改和扩展吗？

假如我现在像增加一个开根号的运算，那么程序应该如何改？是不是需要修改一下Operation类，在其中的switch中加一个分支。这样做确实没问题，但是这样做我们本来就是只要加一个开根的运算，却需要加减乘除的运算都参与编译，再者如果不小心把加法运算改成了减法运算，岂不很糟糕。打个比方，如果现在公司要求张三为公司的薪资管理 系统做维护，原来只有技术人员（月薪），市场销售人员（底薪＋提成），经理（年薪＋股份）三种运算算 法，现在要增加旅职工作人员（时薪）的算法．但按照你昨天的程序写法，公司就必须要把包含原三种 算法的运算类给张三，让张三修改，张三如果心中小算盘一打，'TMD，公司给我的工资这么低，我真是郁闷． 这F有机会了’，于是张三除了增加了兼职算法以外，在技术人员（月薪）算法中写了一句：

if(员工是张三){
    salary = salary * 1.1;
}

那意味着，张三的月薪每个月都会增加10%，本来让他加一个功能，却使得原有的运行良好的宫鞥代码产生了变化，这个风险就很大了。所以我们应该把加减乘除等运算分离，修改其中一个不会影响另外几个，增加运算算法也不影响其他代码，这就要用到面向对象中另外两个特性继承和多态了。

Operation类

abstract class Operation {
    double numA = 0;
    double numB = 0;
    double result = 0;

    public abstract double getResult();

    public double getNumA() {
        return numA;
    }

    public void setNumA(double numA) {
        this.numA = numA;
    }

    public double getNumB() {
        return numB;
    }

    public void setNumB(double numB) {
        this.numB = numB;
    }
}

加减乘除类

public class AddOperation extends Operation {

    @Override
    public double getResult() {
        result = numA + numB;
        return 0;
    }
}

public class SubOperation extends Operation {

    @Override
    public double getResult() {
        result = numA - numB;
        return 0;
    }
}

public class MulOperation extends Operation {

    @Override
    public double getResult() {
        result = numA * numB;
        return result;
    }
}

public class DivOperation extends Operation {

    @Override
    public double getResult() {
        result = numA / numB;
        return result;
    }
}

上面我们首先写了一个抽象的运算类，它有三个属性，分别用来存储运算数和结果，然后是一个抽象方法getResult()用来得到计算结果。之后把加减乘除都写为运算类的子类，继承它后重写getResult()方法，这样如果要修改任何一个算法就不需要提供其他算法的代码了。但我们如何让计算器知道我是希望用哪一个算法呢？接着往下看。

3.3多态&简单工厂模式

接着上文，如何让计算器知道我是希望用哪一个算法呢？这就要用到我们要介绍的“简单工厂模式”了。也就是说，到底要实例化谁（希望用哪一个算法），将来会不会增加实例化对象（比如增加开根号的运算），这些都是很容易变化的地方，应该考虑用一个单独的类来做这个创造实例的过程，这个类就是工厂类。看一下这个类的具体写法：

简单运算工厂类

public class OperationFactory {
    public static Operation createOperation(String operate){
        Operation oper = null;
        switch (operate){
            case "+":
                oper = new AddOperation();
                break;
            case "-":
                oper = new SubOperation();
                break;
            case "*":
                oper = new MulOperation();
                break;
            case "/":
                oper = DivOperation();
                break;
        }
        return oper;
    }
}

这样你只需要输入运算符号，工厂就实例化合适的对象，通过多态，返回父类的方式实现计算器的结果。

客户端代码

Operation oper;
oper = OperationFactory.createOperation("+");
oper.numA = 1;
oper.numB = 2;
double result = oper.getResult();

界面的实现就是这样的代码，不管你是控制台程序，Windows程序，Web程序或手机程序，都可以用这段代码来实现计算器的功能，如果有一天需要改进加法算法，我们只需要修改AddOperation类就可以了，如果需要增加各种复杂的运算，比如平方根、立方根、开根号等只需要增加对应的运算子类，并在工厂类中增加对应的switch分支即可。
